API 自動出票

BE2 系統與供應商訂購平台串接，即時查詢和預訂票券及酒店

自動出票
ℹ️ 定義：API 讓 BE2 系統與供應商訂購平台互相連接並共享資訊，可即時查詢和預訂票券及酒店，大幅減少人工作業需求。
✅ 通常無需 OP 作業，系統自動完成訂購與出票。

⚠️ API 失敗處理方式

❌ OP 會收到兩則錯誤訊息，請依以下步驟處理。
1
Slack 聯繫 BD／AM

透過 Slack 通知對應的 BD 或 AM，說明 API 訂購失敗的訂單資訊。

2
BD／AM 確認失敗原因

BD／AM 與供應商確認 API 失敗原因，可能為庫存不足、連線異常或商品下架等。

3
BD／AM 回覆作業方法

依確認結果，BD／AM 回覆 OP 後續處理方式（可點重串、人工補訂、改期或取消等）。

⚠️ HK AM 補充：HK AM 有建立 API 錯誤訊息對照表，可先查閱確認常見錯誤原因。 ℹ️ 若失敗原因為可疑訂單，風控會協助重新串出 API，OP 無需另外作業。
