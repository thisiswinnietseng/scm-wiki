特例取消流程

異常處理
KKday 正常情況不會主動取消訂單，以下三種情況除外：
1 訂單無法訂出
2 訂單處理完畢後的商品異動
3 供應商因手續費調整請求 KKday 協助取消

🏢 KKday 端取消

點擊特殊取消原因 → 確認手續費 → 備註原因 & 送出給客服 → CS 退款執行
有手續費 → 計算（百分比或定額）→ 選手續費原因 → 備註取消原因 → 送出給客服
無手續費 → 直接送出

👤 客人端需求取消

CS 接收需求 → OP 確認可否取消 → CS 與旅客確認退款條件 → OP Be2 發起取消 → CS 退款執行
OP 確認可否取消時，需聯繫供應商確認同意取消及退費條件
退款依客人付款方式執行原路退款或匯款退款
⚠️ 如有手續費需先告知客人

🏪 供應商端取消（需 OP 作業特例）

適用情況

💰 供應商欲調整取消退費金額
📅 過出發日訂單取消（客訴處理）
✅ 供應商同意取消已核銷訂單

處理流程

供應商通知 OP → OP 確認原因 → Be2 發起特殊原因取消 → CS 與旅客確認 → CS 退款執行

💸 取消手續費情境

✅ 免收手續費
  • 客人在免費取消期間申請取消
  • 供應商疏失引發客訴，供應商同意免費取消
❌ 需收手續費
  • %：e.g. 12/9 前申請取消收 10% 手續費
  • 特定金額：e.g. 公司政策退款，供應商不同意免費取消

🏛️ 公司政策退款（特殊情況）

當客人因特殊情況需取消，但供應商取消政策嚴格無法退款時，客服部門依公司政策審核是否可退款：
😢 旅客二等親內直系血親屬去世
🏥 旅客二等親內直系血親屬生重病
🌪️ 目的地發生嚴重自然災害（不可抗力）
✈️ 原訂班機停飛或嚴重延誤
💡 補充 — API 商品
若商品有串接 API 取消，確認可取消後，進入 Be2 訂單管理的「B2B 串接訂單取消」操作
💡 補充 — PSI 出票
若訂單為 PSI 出票，取消後留意是否需要 PSI 退貨。
注意：無論全額退款（訂單取消）或部分退款（訂單不取消），PSI 訂單的手續費／成本不會變動。
