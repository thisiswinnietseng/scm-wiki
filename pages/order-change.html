訂單更改

依訂單處理單位（供應商自理 / KKday 處理）及旅客聯繫對象分四種情境

異常處理
🏪 供應商自理訂單
旅客聯繫供應商 → 供應商通知 OP
旅客聯繫 KKday → 無（客人直接與 CS 溝通）
🏢 KKday 處理訂單
旅客聯繫供應商 → 供應商通知 OP
旅客聯繫 KKday → 無（客人直接與 CS 溝通）

📝 更改旅客資料 / 規格

🏪 供應商自理 · 旅客聯繫供應商
1
供應商主動通知 OP 更改需求
2
請 CS 領單並齊全，OP 點「已領取」→ OP 改（不需與客人確認）
3
OP 通知供應商，供應商更新憑證並重發給旅客
🏪 供應商自理 · 旅客聯繫 KKday
1
OP 通知供應商 → 供應商同意即可改 → 改之後通知 CS
2
CS 改資料（須與客人確認）
3
OP 通知供應商，供應商更新憑證重發給旅客
🏢 KKday 處理 · 旅客聯繫供應商
1
OP 先確認供應商是否可改
2
供應商同意後 → 請 CS 改資料 → 通知 OP
3
OP 確認無誤後 → 更新憑證並重發給客人
⚠️ 須與客人確認
🏢 KKday 處理 · 旅客聯繫 KKday
1
OP 先確認供應商是否可改
2
供應商同意後 → 請 CS 改資料 → 通知 OP
3
OP 確認無誤後 → 更新憑證並重發給客人
⚠️ 須與客人確認
💡 若供應商不同意更改 → 回覆客服：此單供應商不同意更改，請協助與客告知此單原單安排。

📅 更改出發日期（已過出發日 或 已核銷）

🚫 Be2 無法改單，系統無法更改日期
🏪 供應商自理 · 旅客聯繫供應商
📌 選項 A：單換單
1
詢問供應商是否可單換單
2
供應商同意 → CS 轉達客人 → 客人重新下新單後通知 OP
3
OP 轉達供應商新單號，請供應商取消舊訂單
📌 選項 B：備註在憑證
1
供應商同意 → OP 備註在憑證處，請供應商更新憑證
2
OP 請 CS 通知客人留意新日期
🏪 供應商自理 · 旅客聯繫 KKday
1
OP 聯繫供應商確認是否接受改期
📌 單換單 或 📌 備註在憑證
流程同左欄
🏢 KKday 處理 · 旅客聯繫供應商
📌 單換單 或 📌 備註在憑證
流程同供應商自理左欄
🏢 KKday 處理 · 旅客聯繫 KKday
1
OP 聯繫供應商確認是否接受改期
📌 單換單 或 📌 備註在憑證
流程同供應商自理左欄
💰 若需要補差價
情境 1：供應商建議客人加價給 KKday
1
OP 請 CS 與客人確認是否同意補差價
2
客人同意 → OP 請 BD/AM 開補差價套餐
3
套餐開好後通知 CS，CS 引導客人下單
4
CS 收到差價單後點單並通知 OP
5
OP 通知 BD/AM 下架套餐，通知供應商更新憑證重發旅客
情境 2：供應商建議客人現場支付
1
OP 與供應商確認差價金額
2
請 CS 向客人查詢是否同意
3
客人同意 → OP 轉達供應商並修改訂單，更新憑證並發給客人
💡 若更改後的成本少過原訂單成本 → OP 可開部退退還差價給客人

🗓️ 更改出發日期（未出發 或 未核銷）

🏪 供應商自理 · 旅客聯繫供應商
✅
OP 直接改單
⚠️ 若需加價，依照加價流程或單換單
🏪 供應商自理 · 旅客聯繫 KKday
1
OP 通知供應商 → 供應商同意後 OP 改單
2
改完後通知 CS
⚠️ 若需加價，依照加價流程
🏢 KKday 處理 · 旅客聯繫供應商
1
須與客人確認 → CS 改資料 → OP 通知供應商
⚠️ 若需加價，依照加價流程
🏢 KKday 處理 · 旅客聯繫 KKday
1
OP 確認供應商是否同意改期
2
供應商同意 → OP 更改出發日、更新憑證 → 通知 CS 轉達客人
⚠️ 若需加價，依照加價流程
📋 KKday 處理 — 依訂單狀態
狀態：旅客資料已齊全
改日期 → OP 可直接更改系統，正常出單即可
改旅客資料規格 → 請 CS 直接改系統，正常出單即可
狀態：已訂出，待供應商確認
已確認 → 回到「已處理」處理流程
未確認 → 請 CS 改系統後，更新供應商最新旅客資料，待供應商回覆確認
