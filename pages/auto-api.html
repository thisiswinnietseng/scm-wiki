自動出票 — API & OCBT

API 串接 · 組合商品

🔗 API 自動出票

ℹ️ 定義：API 讓 BE2 系統與供應商訂購平台互相連接並共享資訊，可即時查詢和預訂票券及酒店，大幅減少人工作業需求。
✅ 通常無需 OP 作業，系統自動完成訂購與出票。

⚠️ API 失敗處理方式

❌ OP 會收到兩則錯誤訊息，請依以下步驟處理。
1
Slack 聯繫 BD／AM

透過 Slack 通知對應的 BD 或 AM，說明 API 訂購失敗的訂單資訊。

2
BD／AM 確認失敗原因

BD／AM 與供應商確認 API 失敗原因，可能為庫存不足、連線異常或商品下架等。

3
BD／AM 回覆作業方法

依確認結果，BD／AM 回覆 OP 後續處理方式（可點重串、人工補訂、改期或取消等）。

⚠️ HK AM 補充：HK AM 有建立 API 錯誤訊息對照表，可先查閱對照表確認常見錯誤原因。 ℹ️ 💡 例外：若失敗原因為可疑訂單，風控會協助重新串出 API，OP 無需另外作業。

📦 OCBT 組合商品

ℹ️ 定義：OCBT 由兩家以上不同供應商的即訂即出商品組成，母訂單實際上為空殼訂單，子訂單各自向對應供應商出票。

📋 OCBT 設定限制

  • 母商品前置日必須 ≥ 子商品前置日
  • 月曆模式：指定日
  • 母商品的銷售日期必須符合子商品銷售日期
  • 母商品旅規必須覆蓋子商品旅規
  • 母商品開啟的市場，子商品也要一併開啟市場
⚠️ 當商品無法使用 OCBT，需由 OP 人工作業處理組合商品訂單。

⚠️ OCBT 失敗處理方式

1
Slack 聯繫 BD／AM

透過 Slack 通知對應 BD 或 AM，告知 OCBT 出票失敗的訂單編號。

2
BD／AM 確認失敗原因

BD／AM 確認是哪個子商品出票失敗及原因。

3
BD／AM 回覆後續作業方法

依確認結果，BD／AM 回覆處理方式。

4
若需人工出票

請 BD／AM 提供 ProdOID，至 B2D 頁面進行人工訂出作業。
